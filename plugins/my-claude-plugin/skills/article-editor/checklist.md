# article-editor 挿入判定チェックリスト

各セクションを走査する際に、このチェックリストを参照して編集指示の種別と必要性を判定する。

---

## 判定の基本方針

- **1セクション（H2/H3）につき最大1件**を目安とする
- 複数の種別が該当する場合は**最も効果的なもの1件**を選ぶ
- 明確なトリガーがないセクションには挿入しない（過剰挿入しない）
- 全体として**記事のセクション数の50%以内**に収める

---

## 📷 スクリーンショット・画像の判定基準

以下のいずれかに該当するセクションに挿入する。

### 強い判定（優先度: 高）

- [ ] **番号付き手順リストが3項目以上ある**
  → 手順の実行結果・操作対象を画面で示すと理解が深まる

- [ ] **UI の固有名詞（ボタン名・メニュー名・画面名）が含まれる**
  例: 「〇〇ボタンをクリック」「設定画面を開く」「〇〇タブを選択」
  → 画面上の場所が文字だけでは伝わりにくい

### 補助判定（優先度: 中）

- [ ] **before/after の変化を説明している**
  例: 「変更前は〜、変更後は〜」「設定前と設定後の違い」

- [ ] **「以下のような表示になります」「このような画面が出ます」などの言及がある**

---

## 📊 図表・グラフの判定基準

以下のいずれかに該当するセクションに挿入する。

### 強い判定（優先度: 高）

- [ ] **2つ以上の選択肢・方法・パターンを比較している**
  例: 「〜と〜の違い」「〜vs〜」「〜のメリット・デメリット」
  → 比較表やグラフで視覚的に整理できる

- [ ] **3項目以上の並列要素がある（かつ各項目に説明がある）**
  例: 「3つのパターン」「5つのステップ」「4つの要素」
  → フローチャートや図解で全体像を示せる

### 補助判定（優先度: 中）

- [ ] **割合・数値・期間などの量的な情報がある**
  例: 「3ヶ月で〜」「80%が〜」「月3万円」
  → グラフや図で直感的に伝えられる

- [ ] **プロセス・フロー（〜すると〜になる）を説明している**
  例: 「ステップ1→2→3の流れ」「〜が原因で〜が起きる」

---

## ⚠️ 注意コールアウトの判定基準

以下のいずれかに該当する段落の直前または直後に挿入する。

### 強い判定（優先度: 高）

- [ ] **段落内に「注意」「気をつける」「してはいけない」「NG」「失敗」などの語がある**

- [ ] **禁止・制限・例外を説明している段落**
  例: 「〜の場合は〜できない」「〜に注意が必要」

- [ ] **リスクや副作用を説明している段落**
  例: 「〜をすると〜の問題が起きる」

---

## 💡 ポイント・補足コールアウトの判定基準

以下のいずれかに該当する段落の直前または直後に挿入する。

### 強い判定（優先度: 高）

- [ ] **段落内に「重要」「ポイント」「押さえておく」「覚えておく」「必ず」などの語がある**

- [ ] **著者の経験・一次情報を含む段落（`【一次情報】` タグあり）**
  → 特に価値の高い情報として視覚的に強調する

- [ ] **記事全体の核心となるメッセージを述べている段落**
  例: 「この記事で最も伝えたいことは〜」「結論として〜」

---

## 📌 その他（AI 判断）の判定基準

上記のどの種別にも当てはまらないが、ビジュアル化によって大きく理解が向上する可能性がある場合。

### 判定の観点

- [ ] **同じ概念を2文以上かけて説明しているが、図や表で一目で伝えられそう**

- [ ] **読者が自分でワークシートや記入シートを使うと効果的な箇所**
  例: 「以下の観点で自分の〇〇を書き出してみましょう」

- [ ] **記事全体の構成や流れを俯瞰で示せる箇所（冒頭・まとめなど）**

---

## 挿入しない場合の判断基準

以下の場合は編集指示を挿入しない。

- テキストのみで十分に伝わる内容（抽象的な概念の説明など）
- すでに箇条書きや見出しで整理されており、図表で付加価値が小さい
- 前後のセクションと内容が重複している
- セクション全体の50%超にすでに挿入している場合の追加判定
