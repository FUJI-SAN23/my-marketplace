# ファシリテーター論点整理チェックリスト

article-facilitator が ArticleContext を出力する前に確認するチェックリスト。

---

## Section 1: 情報収集の完了条件

以下の3要素がすべて揃っているか確認する。

- [ ] **テーマ**: 記事の中心トピックが1文で言えるか
- [ ] **想定読者**: 読者の属性・知識レベル・主な悩みが明確か
- [ ] **目的**: 読後に読者に何をしてほしいか / 何を持ち帰ってほしいかが明確か

**判定基準:**
- 3要素すべてに ✅ がついていない場合は情報収集を継続する
- 「なんとなく」「たぶん」など曖昧な状態では完了とみなさない

---

## Section 2: 追加質問の品質確認

- [ ] 1ターンで提示する質問が **2つ以内** に収まっているか
- [ ] テーマ → 読者 → 目的の **優先度順** に質問しているか
- [ ] 推測で補った部分を「〇〇と理解しましたが、正しいですか？」形式で確認しているか
- [ ] 1つの質問で複数のことを聞いていないか（例: 「読者と目的を教えてください」はNG）

---

## Section 3: 一次情報と一般論の区別

- [ ] ユーザーが提供した体験談・メモを **【一次情報】** として記録したか
- [ ] 一次情報と一般的な知識・セオリーが混在していないか
- [ ] ユーザーが提供していない情報を一次情報として扱っていないか
- [ ] 体験談の要約が正確か（意味の改変・誇張がないか）

---

## Section 4: 構成案の品質基準

- [ ] H2 見出しが **3〜5個** の範囲に収まっているか
- [ ] 見出しが「導入 → 本論 → まとめ」の論理的な流れになっているか
- [ ] 最後の H2 が読者の行動につながる締め見出しになっているか
  （例: 「さっそく始める3ステップ」「まとめ: 今日からできること」）
- [ ] 「テーマ」「目的」に対して見出し構成が一致しているか（ずれがないか）
- [ ] 読者の悩みに答える内容が構成に含まれているか

---

## Section 5: ArticleContext 出力前の最終確認

- [ ] ArticleContext の全フィールドに値が入っているか
  - テーマ / 想定読者 / 目的 / 伝えたい要点 / 一次情報 / 文体 / 用途 / 構成案 / 確認日時
- [ ] 構成案の合意をユーザーから明示的に得たか
- [ ] 「です・ます調」または「断定調」など文体が確定しているか
- [ ] ユーザーへの引き継ぎメッセージ（次のステップ案内）を添えているか

---

## Section 6: フェーズゲート確認

- [ ] 構成案の合意なしに本文ドラフト生成を開始していないか
- [ ] 本文生成の要求を受けた場合に構成案確認を先に促したか
- [ ] ArticleContext を出力せずに article-writer への引き継ぎを行っていないか
